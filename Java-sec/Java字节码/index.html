什么是Java字节码

Java非常著名的口号是”Write once, Run anywhere”，这种与平台无关的思想最终由JVM实现，各种不同平台的虚拟机与所以平台都统一使用一种程序存储格式——ByteCode（字节码）。字节码由十六进制组成，JVM以两个十六进制为一组也就是一个字节为单位进行读取，.class文件就是从源代码编译而成的字节码文件：

由于 JVM 规范的存在，只要最终生成了符合 JVM 字节码规范的文件都可以在 JVM 上运行。

利用方法

在Java安全中，一些反序列化方法需要利用到字节码来进行动态类加载进而执行系统命令，那么先来了解一下Java的动态类加载。

动态类加载

首先来了解一些类和方法，ClassLoader 类是 Java 中的一个抽象类，它用于加载类文件。在 Java 中，类文件通常是以字节码的形式存储的，可以通过 ClassLoader 类的子类来加载类文件，这些类被称为加载器。

实际上，所有 Java 应用中的类都是被 java.lang.ClassLoader 类的一系列子类加载的。因此要想动态加载类的话也必须使用 java.lang.ClassLoader 的子类。同时一个类如果被加载到内存，那么与其相关的所有类都会被加载。
