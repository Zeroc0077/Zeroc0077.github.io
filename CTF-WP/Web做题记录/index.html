登录后在源码底端发现：

代码分析：

  • 首先对传入的calc参数进行检查，js中Array的some方法用于检测数组中的元素是否满足指定条件（函数提供），这里的检查逻辑是若calc中出现/(flc'\".中任意一个字符，则这个位置后面全部替换为*；
  • 然后截取calc的前64个字符进行处理，/\s+/g是正则表达式，/ /中间是正则表达式的内容，\s可以匹配任何空白符，g表示全局匹配，就是说替换所有的，若不加g那么匹配到第一个就结束。
  • 然后过滤了sh、ln、fs、x，最后执行eval。

那么这里我们首先需要绕过第一层检测，实际上题目给了提示：字符串的length和数组的length是不一样的

也就是说我们传aaa和['aaaaaa', 'bbbbbbbb', 'cccccccc']两个不同的数据类型，得到的length都是3，那么我们可以这样传参试验一下：calc[]=a(aaaa&calc[]=bbbbbb&calc[]=c(相当于以拼接方式传入数组)：

可以看到我们成功绕过了检测回显出了(字符，这实际上是因为我们传入的calc作为一个数组，它的length=3，所以我们只需要把数组的最后一个元素设置为waf中的一个字符（必须是一个字符，否则没有检测出来的话不会对calc进行字符串拼接转换，所以calc做substring操作时是一个数组object，会报错），那么我们就可以回显length长度的字符串，所以我们需要构造多长的payload，我们的calc数组就要有与payload长度相等的元素个数。

[GKCTF 2020]ez三剑客-easynode

题目源码：

首先分析代码：

  • 访问/eval这个路由时，会先设置一个delay = 60 * 1000，然后与你GET传入的参数delay进行比较，将较大的一个作为参数传入setTimeout(() => next(), delay)这样一个定时器，这里有一个绕过方法：浏览器内部使用32位带符号的整数，来储存推迟执行的时间。这意味着setTimeout最多只能推迟执行2147483647毫秒，超过这个时间会发生溢出，导致回调函数将在当前任务队列结束后立即执行，所以我们只需要传入比2147483647大的delay即可正常访问到/eval这个路由。

  • 接下来在/eval这个路由中调用了这样一个函数response = saferEval(req.body.e);，虽然说是安全的eval，但是是存在漏洞的，而且绕过的手法也不少，详见：https://github.com/commenthol/safer-eval/issues
